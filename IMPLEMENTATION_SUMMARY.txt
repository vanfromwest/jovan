### 9. Database Tables (Feature 28)

#### New Tables Created:
1. **notifications** - Notification messages and management
   - Fields: id, user_id, title, message, type, is_read, is_pinned, action_url, timestamps
   
2. **email_logs** - Email delivery tracking
   - Fields: id, user_id, email, subject, message, status, error_message, sent_at, created_at
   
3. **travel_records** - Faculty travel information
   - Fields: id, faculty_id, destination, purpose, start_date, end_date, total_days, is_active, timestamps
   
4. **email_config** - SMTP and email settings
   - Fields: id, config_key, config_value, updated_at

#### Modified Tables:
1. **faculty_status** - Enhanced for travel support
   - Added: status ENUM now includes 'ON TRAVEL'
   - Added: travel_record_id (foreign key to travel_records)
   
2. **announcements** - Enhanced with priorities
   - Added: is_pinned (Boolean)
   - Added: priority (ENUM: LOW, MEDIUM, HIGH)
   - Added: expiration_date (DateTime)

## Installation Instructions

### Step 1: Database Migration
```
1. Log in as Admin
2. Visit: /setup/migration.php
3. Click "Run Migration"
4. All tables are created automatically
```

### Step 2: Configure Email
```
1. Go to Admin > SMTP Configuration
2. Enter Gmail SMTP credentials
3. Test email delivery
4. Enable email notifications
5. Save configuration
```

### Step 3: Test Features
```
- Approve a user account (email sent)
```
